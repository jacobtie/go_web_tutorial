Go Info

Go is an open source language created by engineers from Google. The language was created by Robert Griesemer, Ken Thompson, and Rob Pike. Go is a compiled, statically typed, and supports garbage collection and concurrency. What makes Go different, is that it was built to be intuitive to develop in, run for the web, and be efficient.

Go is partially object-oriented. It does not have classes and objects in the traditional sense, but instead utilizes a user defined type system, in which developers define complex data types. Functions can be created and bound implicitly to these types so that they function similar to how objects work in other object-oriented languages. Developers define types using the type keyword. A working example of a Go Binary Search Tree as an object can be found here on GitHub. Users may also implement and define interfaces which function similarly to interfaces in other languages.

Go implements parallelism through goroutines. A goroutine fulfills two of the main goals of the Go programming language, those being ease in development and efficiency. Goroutines are incredibly easy to implement; the developer must only write the word go in front of a function call or an anonymous function.

Go was first thought into existence in 2007 by three Google employees, Robert Griesemer, Rob Pike, and Ken Thompson. These three Googlers started designing Go that same year. The primary motivation these three Googlers had for creating Go was their complete dislike of C++. Seemingly fueled by their mutual hatred of C++, the Googlers made Go open-source in 2009, and version 1.0 was released in 2012.

Go was primarily created in order to merge the best parts of multiple programming languages into one, while leaving out the bad parts. Some languages are compiled and fast, like C, but they are unsafe, hard to work with, and can suffer from many runtime errors. Other languages are interpreted, like Ruby. These languages are typically safer but are also slower and have many dependencies. Then you have other languages like Java that require a virtual machine, JVM, in order to run the code, which can make porting the code between different machines unreliable at times.

Go in a general purpose programming language, meaning that it can be used for pretty much anything. However, there are a few things that Go is great for and some things that Go is not very good for.
Go is great for:

  • Command-line apps/scripts.
  • Machine learning.
  • Automation.
  • Networking.
  • Web servers.
Go is not so good for:
  • Desktop/GUI based apps.
  • System level programming.

The creators of Go have worked very hard in order to remove these drawbacks.
Go is:

  • Fast - Go has very fast compilation that you will barely notice.
  • Safe - Go strongly typed, statically typed, and has a garbage collector.
  • Easy - Go prides itself on being concise and explicit.
  • Modern - Has built-in support for many different kinds of applications.
The creators of Go even have idioms for their language:
  • Get the job done.
  • One way of doing things.
  • Be explicit.
  • Build things by composing them.
For all of these reasons, and more, Go has been increasing in popularity over the years, and it does not show any signs of slowing down.
